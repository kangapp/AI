## 严重问题 (Severity: High)

### 1. API Key 验证缺失
**文件**: `project/simple-agent/src/server/routes/agent.ts:30`
**描述**: Agent 实例化时使用 `process.env.ANTHROPIC_API_KEY || process.env.OPENAI_API_KEY || ''` 作为 apiKey，但如果两者都未设置，会传入空字符串，Agent 可能在后续调用时才报错。
**影响**: 请求会成功入队 202，但 Agent 执行时会因空 apiKey 失败，用户体验不佳。
**建议**: 在 Agent 实例化前检查 apiKey 是否有效，无效时返回 400 错误。

```typescript
const apiKey = process.env.ANTHROPIC_API_KEY || process.env.OPENAI_API_KEY;
if (!apiKey) {
  return res.status(400).json({ error: 'API key required. Set ANTHROPIC_API_KEY or OPENAI_API_KEY' });
}
```

## 中等问题 (Severity: Medium)

### 2. 缺少请求体验证
**文件**: `project/simple-agent/src/server/routes/agent.ts:11`
**描述**: `req.body as AgentRunRequest` 直接断言类型，没有验证必填字段 `prompt` 是否存在。
**影响**: 如果请求体缺少 prompt，会导致后续处理出错。
**建议**: 添加基础验证：
```typescript
if (!prompt || typeof prompt !== 'string') {
  return res.status(400).json({ error: 'prompt is required' });
}
```

### 3. DEBUG 日志可能泄露敏感信息
**文件**: `project/simple-agent/src/server/routes/agent.ts:14-18`
**描述**: `/debug/env` 路由返回 `baseURL` 等配置信息。虽然没直接暴露 apiKey，但如果 baseURL 是内部服务地址，可能造成信息泄露。
**影响**: 攻击者可能通过此端点了解系统架构。
**建议**: 
- 生产环境禁用此路由
- 或仅在开发环境暴露
- 考虑移除 baseURL 字段
