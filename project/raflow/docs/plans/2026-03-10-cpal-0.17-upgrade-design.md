# 音频系统升级设计：CPAL 0.17 迁移方案

**日期**: 2026-03-10
**目标**: 修复 macOS open 启动时的音频权限问题

---

## 问题背景

根据 [macOS 音频权限指南](../../docs/macOS-audio-permission-guide.md)，CPAL 0.15 在调用 `default_host()` 时会枚举系统中所有设备的属性，触发麦克风权限弹窗。CPAL v0.17 已修复此问题。

当前项目状态：
- CPAL 版本: 0.15
- macOS 音频后端: `capture_avaudio.rs` (CoreAudio 绕过实现)
- 权限配置: entitlements + NSMicrophoneUsageDescription 已配置
- 冗余代码: 存在 `capture.rs`, `capture_avaudio.rs`, `capture_avfoundation.rs` 三套实现

---

## 设计目标

1. **解决 open 启动问题** - 利用 CPAL 0.17 的官方修复
2. **简化代码架构** - 统一使用 CPAL，移除冗余实现
3. **保持权限策略** - 启动时主动请求麦克风权限

---

## 架构设计

### 1. 依赖升级

```toml
# Cargo.toml
cpal = "0.17"  # 从 0.15 升级
```

### 2. 统一音频后端

所有平台统一使用 CPAL 0.17，移除条件编译的 `capture_avaudio.rs`：

```rust
// mod.rs 简化
mod capture;      // CPAL 实现
mod pipeline;
mod resampler;

pub use capture::*;
pub use pipeline::*;
pub use resampler::*;
```

### 3. 权限请求策略

```rust
impl AudioCapture {
    pub fn new() -> Result<Self, CaptureError> {
        // 1. 主动请求麦克风权限 (macOS)
        #[cfg(target_os = "macos")]
        request_microphone_permission();

        // 2. 使用 CPAL 0.17 初始化
        let host = cpal::default_host();
        let device = host.default_input_device()
            .ok_or(CaptureError::NoInputDevice)?;

        // 3. 构建 StreamConfig
        let config = device.default_input_config()?;

        // ... 创建流
    }
}
```

### 4. 错误处理

| 错误码 | 处理策略 |
|--------|----------|
| `kAudioDevicePermissionsError` | 提示用户在系统设置中开启麦克风权限 |
| `kAudioHardwareBadDeviceError` | 重试 3 次，每次间隔 100ms |
