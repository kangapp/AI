## 测试方法

### 测试驱动开发 (TDD)

所有功能开发都遵循 TDD 流程：

1. 编写测试用例
2. 运行测试（预期失败）
3. 实现最小化代码
4. 运行测试（预期通过）
5. 重构优化
6. 重复

### 性能测试方法

使用自定义的 `Metrics` 和 `Histogram` 实现：

```rust
let metrics = Metrics::new();
{
    let _timer = metrics.timer(MetricType::AudioLatency);
    // 执行操作
}
let snapshot = metrics.snapshot();
// 验证百分位数
```

### 跨平台测试

使用条件编译验证平台特定功能：

```rust
#[cfg(target_os = "macos")]
{
    // macOS 特定实现
}

#[cfg(not(target_os = "macos"))]
{
    // 跨平台备用方案
}
```
